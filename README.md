# KPI Mapper (MVP на 20 показателей)

Простой инструмент, который можно запустить локально и сразу заполнять:
- 20 метрик с кодами `0001..0020`;
- привязка метрик к KPI сотрудников;
- привязка метрик к KPI отделов;
- привязка метрик к верхнеуровневым дашбордам;
- автоматический расчет и базовая визуализация.

## Быстрый запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

## Что заполнять

После первого запуска появятся файлы в `data/`:
- `metrics.csv` — каталог метрик и фактические/плановые значения;
- `employee_mapping.csv` — привязки кодов метрик к KPI сотрудников;
- `department_mapping.csv` — привязки кодов метрик к KPI отделов;
- `dashboard_mapping.csv` — привязки кодов метрик к виджетам дашборда.

Ключ связи везде один: `metric_code`.

## Демо-сценарий (как просили: сразу «живой» пример)

В левом сайдбаре есть кнопка **«Загрузить демо-данные»**. Она заполняет:
- 6 показателей с реалистичными значениями (`0001..0006`),
- 3 сотрудников,
- 2 отдела,
- 1 верхнеуровневый дашборд с 4 виджетами.

После загрузки демо-данных откройте вкладку **«5) Визуализация»** — увидите готовые KPI-таблицы и графики, а затем можете заменить данные на свои.

## Интерфейс

1. **Метрики** — ввод значений и целей.
2. **Привязка к сотрудникам** — какие коды участвуют в KPI конкретного сотрудника.
3. **Привязка к отделам** — агрегирование по отделам.
4. **Привязка к дашбордам** — какие коды идут в верхний уровень.
5. **Визуализация** — таблицы KPI + bar chart.

## Логика расчета (MVP)

- Для `higher_is_better`: `score = value / target * 100`.
- Для `lower_is_better`: `score = target / value * 100`.
- Итоговый KPI = взвешенное среднее по `weight`.

## Аналоги в интернете (для ориентира)

- **Airtable Interfaces** + формулы + представления.
- **Smartsheet** для табличного KPI-трекинга.
- **Power BI Scorecards / Metrics** для управленческих KPI.
- **Tableau** для drill-down (завод → отдел → сотрудник).
- **Geckoboard** для операционных дашбордов в реальном времени.

## Как масштабировать после MVP

- Перевести хранение из CSV в PostgreSQL.
