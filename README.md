# KPI Mapper

## Полная перепроверка и причина "старого экрана"

Сделал полную перепроверку публикации и добавил защиту на всех уровнях. Главные причины, почему вы могли видеть старый экран:
1. Браузер кэшировал старые `styles.css` / `dashboard.js`.
2. GitHub Pages мог быть настроен на `/docs`, а изменения смотрели в root (или наоборот).
3. Jekyll/Pages мог отдавать старую сборку без явного маркера версии.

## Что исправлено сейчас

- Перевел визуал на **новые имена ассетов**:
  - `styles.v2.css?v=20260311b`
  - `dashboard.v2.js?v=20260311b`
- Добавил явный маркер версии в UI: **Build: 2026.03.11-v2**.
- Сделал синхронную публикацию и в root, и в `docs/`:
  - `index.html` + `styles.v2.css` + `dashboard.v2.js`
  - `docs/index.html` + `docs/styles.v2.css` + `docs/dashboard.v2.js`
- Добавил `.nojekyll` и `docs/.nojekyll`.

## Как проверить, что точно открыт новый визуал

1. Откройте:
   - `http://localhost:8080/`
   - `http://localhost:8080/docs/`
2. Убедитесь, что на экране есть надпись:
   - **Build: 2026.03.11-v2**
3. Проверьте функционал:
   - выпадающий список "Набор данных" меняет все KPI,
   - фильтр отдела меняет таблицы/графики,
   - слайдер сценария меняет значения карточек и дашборда.
